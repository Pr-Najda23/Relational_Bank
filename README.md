FinanceCore Database Project 📌 Contexte du projet Ce projet consiste à concevoir une base de données relationnelle normalisée (3NF) dans le domaine bancaire. L’objectif est de structurer, charger et analyser des données financières propres afin de produire des analyses fiables pour le métier. 🎯 Objectifs Concevoir un modèle relationnel cohérent (3NF) Identifier les entités : Clients, Comptes, Transactions, Produits, Agences, Segments Créer une base PostgreSQL avec contraintes d’intégrité Charger des données propres depuis un fichier CSV via SQLAlchemy Réaliser des requêtes SQL analytiques avancées Vérifier l’intégrité des données après insertion 🧱 Étapes du projet 1. 📐 Modélisation Identification des entités principales Création d’un schéma ERD Normalisation en 3NF Définition des PK, FK et contraintes 2. 🗄️ Base de données PostgreSQL Création de la base financecore_db Création des tables avec contraintes : PRIMARY KEY FOREIGN KEY NOT NULL / UNIQUE / CHECK Ajout d’index pour optimiser les performances Création de vues pour simplifier les analyses 3. ⚙️ Chargement des données Utilisation de SQLAlchemy Nettoyage et séparation du fichier financecore_clean.csv Insertion des données dans les tables Gestion des conflits (ON CONFLICT) Vérification de l’intégrité des données 4. 📊 Requêtes SQL analytiques Agrégations (SUM, AVG, GROUP BY, HAVING) Sous-requêtes CASE WHEN (analyse du risque / défaut) Jointures multi-tables Création de vues pour KPIs dashboard
